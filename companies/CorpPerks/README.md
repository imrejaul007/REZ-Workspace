## Overview

CorpPerks is an **Agentic Enterprise Platform** providing a comprehensive Workforce OS for modern organizations. It combines HR management, talent intelligence, employee benefits, and AI-powered HR operations into a unified ecosystem.

### Vision
"One Platform for Work, Growth, Trust & Business"

### Core Capabilities
- **Workforce OS** - Complete HRMS with core HR, payroll, performance, and engagement
- **Talent Intelligence** - AI-powered career development and skill mapping
- **Trust Infrastructure** - Universal identity (CorpID) and compliance
- **Industry Bridges** - BIZORA connects to vertical-specific solutions
- **AI Agents** - 46 specialized AI agents for enterprise tasks

## Deployment Instructions

### Prerequisites
- Node.js 20+
- MongoDB 6+
- Redis 7+
- Docker & Docker Compose

### Local Development

```bash
# Clone repository
git clone https://github.com/imrejaul007/CorpPerks.git
cd CorpPerks

# Install dependencies for all services
npm install

# Start infrastructure (MongoDB, Redis)
docker-compose up -d

# Start API Gateway
cd api-gateway && npm run dev

# Start Backend (new terminal)
cd backend && npm run dev

# Start a web app (new terminal)
cd peopleos && npm run dev
```

### Docker Deployment (Production)

```bash
# Build and start all services
docker-compose -f docker-compose.prod.yml up -d

# Check service status
docker-compose -f docker-compose.prod.yml ps

# View logs
docker-compose -f docker-compose.prod.yml logs -f
```

### Web Apps (Vercel)

```bash
# Deploy PeopleOS
cd peopleos && vercel --prod

# Deploy TalentAI
cd talentai && vercel --prod

# Deploy InsightCampus
cd insight-campus && vercel --prod

# Deploy all web apps
./deploy-all.sh
```

### Mobile Apps (Expo)

```bash
# Configure EAS Build
cd people && eas build:configure

# Build for iOS
cd people && eas build --platform ios

# Build for Android
cd people && eas build --platform android

# Submit to stores
eas submit --platform ios
eas submit --platform android
```

### Environment Variables

Required environment variables are documented in `.env.example`. Copy and configure:

```bash
cp .env.example .env
# Edit .env with your configuration
```

Key variables:
- `MONGODB_URI` - MongoDB connection string
- `REDIS_URL` - Redis connection string
- `JWT_SECRET` - JWT signing secret
- `CORPID_SERVICE_URL` - CorpID service URL
- `API_GATEWAY_URL` - API Gateway URL
