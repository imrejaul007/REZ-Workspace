## PART 8: RUQP - REZ UNIVERSAL QR PROTOCOL

### 8.1 Vision

All QR types should resolve through one unified runtime:

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         RUQP - REZ UNIVERSAL QR PROTOCOL                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Format:                                                                   │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  rez://{type}/{id}?context={}&user={}&time={}                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  Resolution Flow:                                                          │
│                                                                             │
│  1. Parse QR → Extract type, id, context                                  │
│  2. Authenticate user (RABTUL Auth)                                       │
│  3. Fetch QR metadata (owner, permissions)                                 │
│  4. Enrich with context (geo, time, user history)                          │
│  5. Run AI inference (intent, offer, reward)                             │
│  6. Check wallet (balance, tier, offers)                                   │
│  7. Generate personalized response                                         │
│  8. Fire events (attribution, analytics, signals)                         │
│  9. Update graph (connections, relationships)                              │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```
